# Product Requirements Document (PRD)
## 테스트 관리 도구 (Test Management Tool)

**버전:** 1.0.0  
**작성일:** 2026-04-30  
**상태:** Draft  
**담당팀:** 소규모 개발팀 (2~5명)

---

## 1. 개요 (Overview)

### 1.1 제품 비전

QTest, Qase, BrowserStack과 같은 엔터프라이즈 테스트 관리 도구를 벤치마킹하여, 소규모~중규모 팀이 효율적으로 소프트웨어 품질을 관리할 수 있는 웹 기반 테스트 관리 플랫폼을 개발한다.

### 1.2 배경 및 문제 정의

현재 팀 내 테스트 케이스 관리 방식은 스프레드시트나 개별 문서에 의존하고 있어 다음과 같은 문제가 발생하고 있다.

- 테스트 케이스의 버전 관리 및 이력 추적 불가
- 테스트 실행 결과의 체계적인 집계 및 리포팅 부재
- 팀원 간 테스트 진행 현황 공유의 어려움
- 자동화 테스트와 수동 테스트의 통합 관리 불가

### 1.3 목표 (Goals)

| 목표 | 지표 |
|------|------|
| 테스트 케이스 작성 시간 단축 | 기존 대비 40% 감소 |
| 테스트 실행 현황 가시성 확보 | 실시간 대시보드 제공 |
| 팀 협업 효율 향상 | 테스트 담당자 지정 및 리뷰 프로세스 도입 |
| 버그 재현율 향상 | 테스트 케이스-버그 연결 추적 |

### 1.4 범위 (Scope)

**In Scope (1차 개발)**
- 테스트 케이스 CRUD
- 테스트 스위트 계층 관리
- 테스트 실행 및 결과 기록
- 기본 대시보드 및 리포트

**Out of Scope (추후 고려)**
- CI/CD 파이프라인 연동
- 외부 이슈 트래커 연동 (Jira, GitHub Issues 등)
- AI 기반 테스트 케이스 자동 생성
- 모바일 앱

## 2. 사용자 (Users)

### 2.1 주요 사용자 페르소나

**페르소나 1: QA 엔지니어 (주 사용자)**
- 테스트 케이스를 직접 작성하고 실행
- 테스트 결과를 기록하고 버그를 리포트
- 테스트 커버리지 현황을 추적

**페르소나 2: 개발 팀 리드**
- 테스트 현황 대시보드를 통해 품질 지표 모니터링
- 릴리스 준비 여부 판단
- 팀원에게 테스트 작업 할당

**페르소나 3: 일반 개발자**
- 자신이 개발한 기능의 테스트 케이스 확인
- 테스트 실패 원인 파악 및 수정

### 2.2 사용자 여정 (User Journey)

```
테스트 계획 수립 → 테스트 케이스 작성 → 테스트 실행 → 결과 기록 → 리포트 생성
```

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 테스트 케이스 관리

#### 3.1.1 테스트 케이스 CRUD

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 케이스 생성 | 제목, 설명, 전제조건, 테스트 단계, 예상 결과 입력 | P0 |
| 케이스 조회 | 목록 및 상세 보기 | P0 |
| 케이스 수정 | 모든 필드 수정 및 버전 이력 저장 | P0 |
| 케이스 삭제 | 소프트 삭제 (복구 가능) | P1 |
| 케이스 복제 | 기존 케이스를 템플릿으로 재사용 | P1 |

**테스트 케이스 필드 정의**

```
- 제목 (Title) *필수
- 테스트 스위트 (Suite) *필수
- 우선순위 (Priority): Critical / High / Medium / Low
- 타입 (Type): Functional / Regression / Smoke / Performance
- 상태 (Status): Active / Deprecated / Draft
- 전제조건 (Precondition)
- 테스트 단계 (Steps): Step / Expected Result 쌍
- 예상 결과 (Expected Result) *필수
- 담당자 (Assignee)
- 태그 (Tags)
```

#### 3.1.2 테스트 스위트 관리

- 폴더 구조의 계층형 테스트 스위트 구성 (최대 5 depth)
- 드래그 앤 드롭으로 케이스 이동
- 스위트 단위 테스트 실행

### 3.2 테스트 플랜 (Test Plan)

테스트 플랜은 릴리스 일정에 기반한 테스트 실행 계획을 관리한다. 마일스톤으로 테스트 단계를 분할하고, 환경별 테스트 범위를 정의할 수 있다.

#### 3.2.1 플랜 기본 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 플랜 생성 | 플랜명, 설명, 배포 예정일 입력 | P0 |
| 플랜 조회 | 목록 및 상세 보기 (상태별 필터링) | P0 |
| 플랜 수정 | 모든 필드 수정 가능 | P0 |
| 플랜 삭제 | 소프트 삭제 (복구 가능) | P1 |
| 플랜 복제 | 기존 플랜을 템플릿으로 재사용 | P1 |

**테스트 플랜 필드 정의**

```
- 플랜명 (Name) *필수
- 설명 (Description)
- 배포 예정일 (Release Date) *필수 (D-day 기준점)
- 담당자 (Owner) *필수
- 상태 (Status): 준비중 / 진행중 / 완료 / 취소
- 생성일 / 수정일 (자동 기록)
```

#### 3.2.2 마일스톤 관리

마일스톤은 배포일을 기준으로 테스트 단계를 구분한다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 마일스톤 생성 | 마일스톤명, 목표 날짜 입력 | P0 |
| 마일스톤 편집 | 날짜 및 설명 수정 | P0 |
| 배포일 마일스톤 | 배포 예정일은 고정 마일스톤으로 자동 생성 | P0 |
| 마일스톤 진행률 | 마일스톤별 테스트 케이스 완료율 통계 | P1 |
| 타임라인 뷰 | 마일스톤을 시간축 기반 타임라인으로 시각화 | P2 |

**마일스톤 예시**

```
┌─ 플랜: v2.0 정식 출시
│
├─ 2026-05-15 - [Smoke Test] 기본 기능 검증
├─ 2026-05-22 - [Functional Test] 주요 기능 안정성 테스트
├─ 2026-05-29 - [Regression Test] 기존 기능 호환성 검증
└─ 2026-06-05 - [Release Day] 배포 (고정 마일스톤)
```

#### 3.2.3 환경 파라미터 선택

플랜에 포함된 모든 테스트 실행에 적용될 환경 설정을 선택한다 (멀티 선택 지원).

| 파라미터 | 선택지 | 기본값 |
|---------|--------|--------|
| **브라우저** | Chrome / Firefox / Safari / Edge | Chrome 선택 |
| **운영체제** | Windows / macOS / Ubuntu / iOS / Android | Windows 선택 |
| **디바이스** | Desktop / Mobile / Tablet | Desktop 선택 |
| **해상도** | 1920×1080 / 1440×900 / 1366×768 / 1024×768 / 768×1024 / 375×812 / 414×896 등 커스텀 입력 지원 | 1920×1080 |
| **네트워크** | 4G / LTE / WiFi / Offline (추후) | 고정값: WiFi 및 4G |

**파라미터 조합 예시**

```
✓ Chrome + Windows + Desktop + 1920×1080
✓ Safari + macOS + Desktop + 1440×900
✓ Chrome + iOS + Mobile + 375×812
✓ Firefox + Ubuntu + Desktop + 1024×768
```

사용자는 **다중 조합을 선택**하여 각 조합별로 테스트 케이스를 실행할 수 있다.

#### 3.2.4 테스트 스위트 연결

플랜에 포함할 테스트 스위트를 선택한다 (체크박스 형식).

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 스위트 선택 | 한 개 이상의 스위트 선택 가능 | P0 |
| 계층 선택 | 상위 스위트 선택 시 하위 케이스 자동 포함 | P0 |
| 선택된 케이스 수 | 플랜에 포함될 케이스 수 실시간 표시 | P1 |
| 선택 초기화 | 선택 해제 버튼 | P1 |

#### 3.2.5 플랜 상세 뷰

플랜 생성 후 확인하는 상세 뷰에서는 다음 정보를 표시한다.

| 항목 | 설명 |
|------|------|
| 마일스톤 타임라인 | 모든 마일스톤을 시간축 기반으로 시각화 (현재 날짜 표시) |
| 진행 현황 | "5 / 20 완료" 형식의 전체 진행률 |
| 마일스톤별 진행률 | 각 마일스톤의 테스트 실행 현황 |
| 환경별 적용 현황 | 선택된 환경 파라미터 목록 표시 |
| 관련 테스트 런 | 이 플랜에 링크된 모든 테스트 런 목록 |

### 3.3 테스트 실행 (Test Run)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 실행 생성 | 테스트 플랜에 연결된 스위트/케이스를 선택해 테스트 런 생성 | P0 |
| 환경 할당 | 테스트 런 생성 시 특정 환경 (브라우저, OS 등)을 할당 | P0 |
| 결과 기록 | Passed / Failed / Blocked / Skipped 상태 기록 | P0 |
| 실패 상세 기록 | 실패 원인, 스크린샷 첨부, 환경 정보 자동 기록 | P0 |
| 실행 이력 조회 | 과거 실행 이력 및 결과 비교 | P1 |
| 진행률 추적 | 실시간 실행 진행 현황 표시 | P1 |
| 환경별 결과 집계 | 같은 케이스를 여러 환경에서 실행할 때 환경별 결과 비교 | P2 |

### 3.4 대시보드 및 리포트

| 커버리지 리포트 | 기능별 테스트 커버리지 현황 | P2 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 성능

- 페이지 초기 로드: 2초 이내
- 테스트 케이스 목록 로드 (1,000건): 1초 이내
- 동시 접속 사용자: 최소 50명 지원

### 4.2 브라우저 지원

| 브라우저 | 최소 버전 |
|----------|----------|
| Chrome | 최신 2개 버전 |
| Firefox | 최신 2개 버전 |
| Safari | 최신 2개 버전 |
| Edge | 최신 2개 버전 |

### 4.3 보안

- HTTPS 통신 필수
- 세션 기반 인증 (JWT)
- RBAC (역할 기반 접근 제어)
- 입력값 XSS/SQL Injection 방어

### 4.4 접근성

- WCAG 2.1 AA 기준 준수
- 키보드 네비게이션 지원

## 6. 화면 구성 (Screen Architecture)

```
📁 테스트 관리 도구
│
├── 🔐 인증 (비로그인)
│   ├── 로그인
│   │   ├── Google OAuth 2.0 연동
│   │   └── 카카오톡 로그인 연동
│   └── 최초 가입 온보딩 (프로젝트 생성 유도)
│
├── 📂 프로젝트 관리
│   ├── 프로젝트 목록 (카드 형태 조회)
│   └── 프로젝트 생성 (이름, 설명, 멤버 초대)
│
│   ── 이하 프로젝트 선택 후 진입 ──
│
├── 🏠 대시보드
│   ├── 전체 테스트 케이스 현황 (총 수, 상태별 분포)
│   ├── 최근 테스트 실행 결과 요약
│   ├── Pass / Fail / Blocked / Skipped 비율 차트
│   ├── 기간별 테스트 결과 트렌드 (꺾은선 그래프)
│   ├── 스위트별 커버리지 현황 (히트맵 또는 바 차트)
│   ├── 담당자별 실행 현황
│   ├── 최근 실패 케이스 목록 (바로가기)
│   └── 미실행 케이스 현황
│
├── 📋 테스트 케이스
│   ├── 스위트 트리 (좌측 사이드바, 계층 구조)
│   ├── 케이스 목록 (필터, 정렬, 검색)
│   │   └── 케이스 클릭 시 → 우측 디테일 패널 슬라이드 인 (페이지 전환 없음)
│   ├── 케이스 디테일 패널 (Right Side Panel)
│   │   ├── 패널 열림: 목록에서 케이스 클릭 시 오른쪽에서 슬라이드 인
│   │   ├── 패널 너비: 480px (기본) / 전체화면 확장 버튼 제공
│   │   ├── 목록과 패널 동시 노출 (ClickUp Task Detail 패턴)
│   │   ├── 조회 모드: 제목, 상태, 우선순위, 담당자, 태그, 스텝 등 표시
│   │   ├── 인라인 편집: 각 필드 클릭 시 즉시 편집 가능 (저장 버튼 없이 자동 저장)
│   │   ├── 상단 액션 바: 편집 / 복제 / 삭제 / 전체화면 / 닫기
│   │   └── 패널 닫기: X 버튼 또는 ESC 키, 목록 빈 영역 클릭
│   ├── 케이스 생성: 우측 패널을 빈 상태로 열어 신규 입력
│   ├── Excel 업로드 (일괄 케이스 등록)
│   └── 테스트 스위트 관리 (생성 / 편집 / 삭제 / 이동)
│
├── 📋 테스트 플랜
│   ├── 플랜 목록 (상태: 준비중 / 진행중 / 완료)
│   ├── 플랜 생성 / 편집

│   │   │   ├── 마일스톤 추가 / 삭제 (이름 + 날짜)
```
